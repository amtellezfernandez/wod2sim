# Diagnostic Protocol Input

The contract-validation matrix (CVM) diagnostic experiment uses
`diagnostic_protocol_sessions.jsonl` as its current-schema input. The file is
generated by:

```bash
uv run python scripts/run_diagnostic_experiment.py
```

The generator instantiates 15 separate dependency-light
`WOD2SimChallengeAdapter` sessions. Together they contain 405 telemetry events,
120 drive calls, and 120 explicit finite serialized-output records under
`wod2sim_challenge_telemetry_v2`. Camera aliases, route shapes, speeds, and valid
sensor ages vary across sessions.

This input is generated protocol-conformance evidence, not an external AlpaSim
rollout. The retained evaluator-owned version-one trace under
`artifacts/external/alpasim_e2e_challenge_conformance` remains separate
interface-conformance evidence and is not used for the mutation experiment.

Generation timestamps, measured call latencies, and the resulting file hash may
change when the experiment is rerun. The experiment validates the complete
trace, requires only the current telemetry schema, and rejects missing session
identity, unknown events, absent finite-output evidence, or any other clean-trace
contract violation before constructing fault/control pairs.
